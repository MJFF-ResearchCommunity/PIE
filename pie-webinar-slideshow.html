What is PIE?

Parkinson's Insight Engine: Your Research Accelerator

"From Raw Data to Research Insights in a Single Command"

PIE is an open-source, automated machine learning pipeline specifically designed for Parkinson's researchers working with PPMI data.

Key Principles:

  1. Researcher-Friendly: No ML expertise required
  2. Transparent: Every step documented and explainable
  3. Reproducible: Consistent results across research groups
  4. Comprehensive: Handles the full ML workflow
  5. Parkinson's-Specific: Built for our unique challenges

The Challenge of PPMI Data

What Makes PPMI Data So Complex?

1. Scale and Scope

📊 Subject Characteristics: Demographics, genetics, family history 🧬 Biospecimens: 12+ projects, proteomics, metabolomics 🧠 Motor Assessments: MDS-UPDRS, detailed clinical evaluations 🎯 Non-Motor: Cognitive tests, sleep, autonomic function 📈 Medical History: Medications, comorbidities, procedures

2. Data Quality Issues

  • Missing Data: Up to 80% missingness in some biomarker columns
  • Inconsistent Formats: Mixed data types, encoding variations
  • Temporal Complexity: Different visit schedules across cohorts
  • Feature Explosion: 10,000+ potential features after preprocessing

The Traditional Approach

Months of Manual Work

# What researchers typically face: 1. Download all the PPMI data from LONI 2. Spend weeks understanding data structure 3. Write custom loading scripts for each modality 4. Handle missing data inconsistently 5. Merge tables (and lose data in the process) 6. Engineer features manually 7. Select features using ad-hoc methods 8. Train models with basic approaches 9. Generate basic plots 10. Write up results (often non-reproducible) Total time: 3-6 months per analysis

There has to be a better way...

PIE's Solution: One Command

Complete Analysis in 30-60 Minutes

python3 pie/pipeline.py \ --data-dir ./PPMI \ --output-dir ./output/my_research \ --target-column COHORT \ --leakage-features-path config/leakage_features.txt \ --fs-method fdr \ --fs-param 0.05 \ --n-models 5 \ --tune \ --budget 60.0

Result: Complete analysis in 30-60 minutes instead of months!

The Four-Stage Pipeline

Stage 1: Data Reduction

  • Loads all PPMI modalities automatically
  • Analyzes data quality before merging
  • Removes low-value features
  • Consolidates into single dataset
  • Generates detailed report

Stage 2: Feature Engineering

  • One-hot encoding for categoricals
  • Numeric scaling (standard/min-max)
  • Polynomial features for interactions
  • Special format handling
  • Custom transformations

Stage 3: Feature Selection

  • False Discovery Rate (FDR) control
  • K-best selection with CV
  • Prevents overfitting
  • Maintains interpretability

Stage 4: Automated Model Comparison

  • Multiple algorithms tested
  • Hyperparameter tuning
  • Cross-validation
  • SHAP analysis
  • Comprehensive visualizations

Technical Architecture

Modular and Extensible Design

pie/ ├── data_loader.py # Multi-modal PPMI data loading ├── data_reducer.py # Data reduction ├── feature_engineer.py # Feature engineering ├── feature_selector.py # Feature selection ├── classifier.py # Automated ML ├── pipeline.py # Orchestrates entire workflow └── reporting.py # HTML report generation

Key Assumption: The Patient-Visit Snapshot

PIE restructures scattered PPMI data into a unified format where each row represents one patient at one visit, with all relevant data in a single row. This is fundamental for machine learning and data consistency.

Choosing the Right Model in PIE

Feature sets for cohort classification

Model‑A: Clinical & Questionnaire

  • Demographics, identifiers, family history
  • Medical history, neurological exam, vital signs
  • MDS‑UPDRS Parts I–III item‑level scores; Part IV flags only
  • Cognitive, sleep, mood, autonomic batteries (MoCA, HVLT, RBDSQ, SCOPA‑AUT, etc.)
  • Includes prodromal flags ENRLHPSM (hyposmia) and ENRLRBD (RBD)
  • Excludes lab assays and genetics

Model‑B: Objective Biomarkers

  • Genomics: all variant columns and enrolment mutation flags
  • Proteomics: biospecimen_project_177_* protein abundances
  • CSF/plasma/serum assays (α‑syn, NfL, GFAP, Aβ₁₋₄₂, lipids)
  • Blood chemistry & haematology indices
  • Excludes clinical/questionnaire content

Model‑C: Combined

  • Union of Model‑A and Model‑B features
  • Often best accuracy; interpret with care

Notes & Exclusions

  • Remove study‑design artefacts (visit counts, sub‑study flags, logistics timestamps)
  • Exclude treatment‑dependent variables to avoid label leakage (e.g., medication dynamics, UPDRS Part‑IV totals)
  • Keep ENRLHPSM and ENRLRBD in Model‑A as prodromal phenomena
  • For genetics (Model‑B), include ancestry PCs when cohorts are ancestry‑mixed
  • Drop non‑informative biospecimen logistics flags unless numeric concentrations

Getting Started with PIE

Quick Start Guide

1. Prerequisites

  • Python 3.8+
  • 8GB+ RAM (16GB recommended)
  • 10GB+ disk space
  • PPMI data access from ppmi-info.org

2. Installation

git clone https://github.com/MJFF-ResearchCommunity/PIE.git cd PIE pip install -r requirements.txt pip install -e .

3. Your First Analysis

python3 pie/pipeline.py \ --data-dir ./PPMI \ --output-dir ./output/my_first_analysis \ --target-column COHORT \ --fs-method fdr \ --n-models 3

Q&A Session

Frequently Asked Questions - Part 1

Q: How does PIE handle different PPMI data versions?

A: PIE is designed to be robust to PPMI data structure changes. The data loaders automatically detect column variations and handle missing files gracefully. We regularly update PIE to support new PPMI releases.

Q: Can PIE be used for other neurodegenerative diseases?

A: While PIE is optimized for PPMI data, the core architecture is modular and extensible. We're actively working on adaptations for Alzheimer's (ADNI), ALS, and other neurodegenerative disease datasets.

Q: What about regulatory compliance for clinical use?

A: Current PIE is designed for research use. We're developing PIE Clinical with FDA/EMA compliance features, including audit trails, validation protocols, and clinical decision support interfaces.

Frequently Asked Questions - Part 2

Q: How do I cite PIE in my publications?

Hamilton, C., & Catterson, V. (2024). PIE: Parkinson's Insight Engine - An Automated Machine Learning Pipeline for PPMI Data Analysis. GitHub: https://github.com/MJFF-ResearchCommunity/PIE

Q: Can I contribute my own analysis modules?

A: Absolutely! PIE is open-source and welcomes contributions. See our GitHub repository for contribution guidelines and development documentation.

Q: What's the difference between PIE and other AutoML tools?

A: PIE is specifically designed for Parkinson's research and PPMI data. Unlike general AutoML tools, PIE understands the unique challenges of multi-modal longitudinal clinical data and includes domain-specific features like data leakage prevention for clinical variables.
